Advanced Query Example

;WITH DailyAverages AS (
    SELECT 
        s.SymbolName,
        s.Sector,
        CAST(t.TimeStamp AS DATE) AS TradeDate,
        COUNT(*) AS TradeCount,
        AVG(t.[Open]) AS AvgOpen,
        AVG(t.[Close]) AS AvgClose,
        MAX(t.[High]) - MIN(t.[Low]) AS DailyRange,
        SUM(t.Volume) AS TotalVolume
    FROM dbo.TradingDatas t
    INNER JOIN dbo.Symbols s ON t.SymbolId = s.SymbolId
    WHERE t.Volume > 100000
    GROUP BY s.SymbolName, s.Sector, CAST(t.TimeStamp AS DATE)
)
SELECT 
    *,
    RANK() OVER (PARTITION BY Sector ORDER BY TotalVolume DESC) AS VolumeRank
FROM DailyAverages
ORDER BY Sector, TotalVolume DESC;

Financial Transaction Example (SQL)

BEGIN TRANSACTION;

-- 1. Debit from user's cash balance
UPDATE accounts
SET balance = balance - 500.00
WHERE account_id = 101 AND balance >= 500.00;

IF @@ROWCOUNT = 0
BEGIN
    ROLLBACK TRANSACTION;
    RAISERROR ('Insufficient funds', 16, 1);
    RETURN;
END

-- 2. Credit user's securities holdings
INSERT INTO holdings (account_id, security_id, quantity, avg_price)

-- 3. Record the trade

COMMIT TRANSACTION;

üîê Security Best Practices

üîê Secure Design and Architecture

  • Adopt Security by Design mindset - bake security into every stage
  • Use threat modeling (STRIDE) to identify attack vectors early
  • Segment architecture: separate public, application, and data layers
  • Principle of Least Privilege (PoLP): Minimum required access
  • Zero Trust Architecture: Continuously verify identity and device trust

üîë Authentication and Authorization

  • Use OAuth 2.1, OpenID Connect for user sessions
  • JWTs with short expiry for API calls
  • Enforce multi-factor authentication (MFA) for privileged accounts
  • Implement RBAC (Role-Based) or ABAC (Attribute-Based) access control
  • Rotate keys, tokens, credentials regularly
  • Use secrets manager (Azure Key Vault, AWS Secrets Manager, HashiCorp Vault)
  • Never hardcode secrets in code or config

üß± Data Protection

In Transit

  • TLS 1.2+ (ideally TLS 1.3)
  • HTTPS everywhere
  • Certificate pinning

At Rest

  • AES-256 encryption
  • Managed or customer-managed keys
  • Field-level encryption for PII

Password Security

  • Hash with salt using bcrypt, Argon2, or PBKDF2
  • Never store plain-text passwords
  • Data minimization - avoid storing unnecessary PII

Git

Common Git Commands

  • git init - initialize a new repository
  • git clone <url> - clone remote repo
  • git status - show working tree status
  • git add <file> - stage changes
  • git commit -m "msg" - commit staged changes
  • git push - push commits to remote
  • git pull - fetch + merge from remote
  • git fetch - download objects and refs
  • git log - view commit history
  • git diff - show changes

Branching & Merging

  • git branch - list branches
  • git branch <name> - create branch
  • git checkout <branch> - switch branch
  • git switch -c <name> - create & switch (modern)
  • git merge <branch> - merge branch into current
  • git rebase <base> - reapply commits onto base (cleaner history)

Remote Workflows

  • git remote -v - list remotes
  • git remote add origin <url> - add remote
  • git push -u origin <branch> - push and set upstream
  • git pull --rebase - rebase local changes onto remote

Undoing & Recovering

  • git restore <file> - discard working-tree changes (modern)
  • git restore --staged <file> - unstage file
  • git reset --soft HEAD~1 - keep changes, move HEAD back
  • git reset --hard HEAD~1 - remove last commit and working changes
  • git revert <commit> - create a new commit that undoes a commit

Stashing, Cherry-pick & Tags

  • git stash - save uncommitted changes
  • git stash pop - apply and remove stash
  • git cherry-pick <commit> - apply single commit onto current branch
  • git tag -a v1.0 -m "release" - create annotated tag
  • git push origin --tags - push tags to remote

Inspecting & Debugging

  • git blame <file> - show who changed each line
  • git bisect start - binary search for faulty commit
  • git show <commit> - show commit details
  • git reflog - view reference log to recover lost commits

Config & Best Practices

  • git config --global user.name "Name" - set identity
  • git config --global core.editor "code --wait" - set default editor
  • Use feature branches, small commits, descriptive messages, and PR-based reviews
  • Prefer git pull --rebase or rebasing local changes to keep history linear
