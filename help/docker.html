                    Common Docker Commands (With Descriptions)
1. docker version

Shows client and server version information.

2. docker info

Displays system-wide Docker configuration and stats.

3. docker pull 

Downloads an image from a registry, usually Docker Hub.

4. docker images

Lists all images stored locally.

5. docker rmi 

Removes a local image.

6. docker run 

Creates and starts a new container from an image.

7. docker run -d 

Runs a container in the background (detached mode).

8. docker run -p host:container 

Maps ports from container to host, useful for exposing apps.

9. docker run --name  

Starts a container with a specific name.

10. docker ps

Shows running containers.

11. docker ps -a

Shows all containers, including stopped ones.

12. docker stop 

Stops a running container.

13. docker start 

Starts an existing stopped container.

14. docker restart 

Restarts a container.

15. docker rm 

Deletes a stopped container.

16. docker logs 

Shows a containerâ€™s logs.

17. docker exec -it  sh or bash

Opens an interactive shell inside a running container.

18. docker build -t name:tag .

Builds an image from a Dockerfile in the current directory.

19. docker tag  

Adds a new tag to an image before pushing it.

20. docker push 

Uploads an image to a registry.

21. docker network ls

Lists networks.

22. docker network create 

Creates a new custom network.

23. docker volume ls

Lists volumes.

24. docker volume create 

Creates a named volume.

25. docker compose up

Starts all services defined in a docker-compose.yml file.

26. docker compose up -d

Starts the services in detached mode.

27. docker compose down

Stops and removes services, networks, and containers from the compose project.

28. docker inspect 

Shows detailed JSON metadata about a container or image.

29. docker cp container:path
 

Copies files from/to a container.

30. docker stats

Shows live resource usage of running containers.
