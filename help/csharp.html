// Filter products by price

    var expensiveProducts = products
    .Where(p => p.Price > 100)
    .ToList();

// Transform to different type
var productNames = products
    .Select(p => p.Name)
    .ToList();

// Chain multiple operations
var result = products
    .Where(p => p.Price > 50 && p.InStock)
    .OrderBy(p => p.Name)
    .Select(p => new { p.Name, p.Price })
    .ToList();

// Group by category
var groupedProducts = products
    .GroupBy(p => p.Category)
    .Select(g => new {
        Category = g.Key,
        Count = g.Count(),
        AvgPrice = g.Average(p => p.Price)
    });

// Check conditions
bool hasExpensiveItems = products.Any(p => p.Price > 1000);
bool allInStock = products.All(p => p.InStock);

// Join two collections
var orderDetails = orders
    .Join(customers,
        order => order.CustomerId,
        customer => customer.Id,
        (order, customer) => new {
            OrderId = order.Id,
            CustomerName = customer.Name,
            Total = order.Total
        });

// Aggregate operations
var totalRevenue = orders.Sum(o => o.Total);
var avgOrderValue = orders.Average(o => o.Total);
var maxOrder = orders.Max(o => o.Total);

// Skip and Take (pagination)
var page = products
    .OrderBy(p => p.Name)
    .Skip(20)
    .Take(10)
    .ToList();

// Distinct values
var uniqueCategories = products
    .Select(p => p.Category)
    .Distinct()
    .ToList();

üì¶ Package Manager (NuGet)

Overview

NuGet is the package manager for .NET. Use the modern dotnet CLI with PackageReference in .csproj, or nuget.exe for legacy scenarios and publishing. Support for private feeds via nuget.config.

dotnet CLI - Common Commands

Command Description / Example
dotnet add package <PackageId> Add package to project (resolves latest stable)
dotnet add package <Pkg> --version 1.2.3 Add specific version
dotnet restore Restore packages for solution/project
dotnet list package List installed packages and available updates
dotnet pack Create a NuGet package from a project
dotnet nuget push <*.nupkg> --api-key <key> --source <url> Push package to feed (NuGet.org or private)
dotnet tool install -g <tool> Install global .NET tool

nuget.exe - Useful Commands

nuget.exe restore MySolution.sln
nuget.exe pack MyProject.nuspec -Version 1.0.0
nuget.exe push MyPackage.1.0.0.nupkg -Source "https://api.nuget.org/v3/index.json" -ApiKey <key>
nuget.exe locals all -clear  -- clear local cache

PackageReference (.csproj) Example

<ItemGroup>
<PackageReference Include="Newtonsoft.Json" Version="13.0.1" />
<PackageReference Include="Polly" Version="7.2.3" />
</ItemGroup>

nuget.config - Private Feed Example

                    <configuration>
                  <packageSources>
                    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
                    <add key="MyPrivateFeed" value="https://pkgs.example.com/nuget" />
                  </packageSources>
                  <packageSourceCredentials>
                    <MyPrivateFeed>
                      <add key="Username" value="user" />
                      <add key="ClearTextPassword" value="secret" />
                    </MyPrivateFeed>
                  </packageSourceCredentials>
                </configuration>

Versioning & Pre-release

  • Semantic versions: 1.2.3, use ranges in <PackageReference> if needed.
  • Pre-release identifiers: 1.2.3-beta.1 ‚Äî must allow pre-release when installing.
  • Floating versions: 1.2.* (use cautiously).

Tools & Best Practices

  • Prefer PackageReference (transitive deps, cleaner restore).
  • Pin versions for production-critical dependencies; use updates via CI checks.
  • Use private feeds for internal packages and CI artifacts.
  • Scan packages for vulnerabilities (Dependabot, Snyk, GitHub Advanced Security).
  • Cache restore in CI (use dotnet restore with --runtime or restore caches).
  • Sign important packages and verify package integrity where required.
